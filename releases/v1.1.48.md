## English

### Fixed

- **MongoDB indexes** — Dynamic API syncs schema indexes on startup (background, after `/api/health` is up) so production updates apply new indexes without manual `mongosh`
- **Telemetry queries** — compound index `{ resourcePath, postSerial, messageType, receivedAt }` speeds filtered list/count on post detail and MQTT log

### Upgrade

Auto-update from Dashboard (**Settings → Updates**) or:

```bash
git pull
docker compose build dynamic-api dashboard
docker compose up -d dynamic-api dashboard
```

First start after upgrade may build the new index in the background (large telemetry volumes); CRM stays available.

## Русский

### Исправлено

- **Индексы MongoDB** — Dynamic API синхронизирует индексы из схем при старте (в фоне, после поднятия `/api/health`), чтобы при обновлении production не требовался ручной `mongosh`
- **Запросы телеметрии** — составной индекс `{ resourcePath, postSerial, messageType, receivedAt }` ускоряет фильтрованные списки и count на странице поста и в журнале MQTT

### Обновление

Автообновление из Dashboard (**Настройки → Обновления**) или:

```bash
git pull
docker compose build dynamic-api dashboard
docker compose up -d dynamic-api dashboard
```

После обновления новый индекс может строиться в фоне (при большом объёме телеметрии); CRM остаётся доступным.
