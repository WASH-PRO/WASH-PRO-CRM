**Жарық**, **қараңғы** және **жүйелік** тақырыпты (ОС баптауларын қайта жүктеусіз қадағалайды), бекітілген бүйір мәзірі, **live-жаңарту** және екі бағанды кіру беті бар React-қосымша (макет PyOrchestrator сияқты, өз фоны).

## Навигация

| Топ | Бөлімдер |
|-----|----------|
| **Негізгі** | Шолу, **Күй** (барлық посттар, график, онлайн/оффлайн) |
| **Объектілер** | Автомойкалар, Посттар |
| **Деректер** | MQTT |
| **Карталар** | Жеңілдік, Сервистік, VIP, Инкассация |
| **Аналитика** | Пайдалану статистикасы, Қаржы статистикасы, Архивтеу |
| **Анықтамалықтар** | Жұмыс режимдері, Валюталар, Жеңілдік түрлері *(Admin)* |
| **Автоматтандыру** | **Ақпарат**, **Telegram**, **MCP сервер**, Резервтік көшірмелер *(Admin)* |
| **Жүйе** | Хабарландырулар, **Пайдаланушылар**, **Топтар мен құқықтар**, **Баптаулар**, Логтар *(Admin)* |

admin белгісі бар бөлімдер тек **Administrator** үшін (`manage_users` немесе `view_logs` JWT-де).  
**Профиль** — тақырыптағы сілтеме (`/profile`). **Баптау шебері** — `/setup` (бірінші кіруде немесе `?restart=1`).

### Resources (сайдбар)

| Элемент | Сипаттама |
|---------|----------|
| **Dynamic API** | Панель `:8080`, online/offline күйі |
| **PyOrchestrator** | Панель `:8090`, bridge health арқылы күй |
| **Құжаттама / GitHub** | Сыртқы сілтемелер |

Кіріктірілген платформалар туралы толығырақ: [Кіріктірілген сервистер](embedded-services.md).

## Live-режим

Оперативті деректер автоматты жаңартылады (`usePolling`). Тақырыпта live-режим индикаторы.

| Интервал | Беттер |
|----------|--------|
| 3 с | Посттардың ағымдағы күйі, **Посттар**, **пост карточкасы** |
| 5 с | Шолу |
| 10 с | Карталар, қаржы, пайдалану, хабарландырулар |
| 15 с | Автомойкалар, архив, бэкаптар, валюталар, жеңілдік түрлері, логтар, пайдаланушылар, топтар, баптау шебері |

**Ағымдағы күй** бетінде режим уақыты нақты уақытта тикады (`LiveModeTimer`).

## Кестелер (DataTable)

CRM-нің негізгі кестелері ортақ **DataTable** компонентін біртұтас беттеумен қолданады.

### Мүмкіндіктер

- мәтіндік өрістер бойынша іздеу;
- тақырыпқа басу арқылы сұрыптау;
- сүзгілер (ашылмалы тізімдер);
- **көрінетін бағандарды таңдау** (кесте баптаулары белгішесі);
- **беттеу** (төменге қараңыз);
- жаппай әрекеттер (CSV экспорт, жою — қолжетімді болғанда).

### Беттеу (v1.1.5+)

**Әр кестенің төменгі бөлігінде**:

| Элемент | Сипаттама |
|---------|----------|
| **Бетте** | Ашылмалы тізім: **20 / 40 / 60 / 80 / 100** жазба (әдепкі **20**) |
| **Артқа / Алға** | Жүктелген жолдар арасында беттер арасында өту |
| **Тағы жүктеу (N жазба)** | Сүзілген жиыннан келесі порцияны жүктеу (N = таңдалған бет өлшемі) |

Төменгі бөліктегі жазба: `N жазба · бет X / Y · жүктелді M` — ағымдағы деректер жиынында әлі көрсетілмеген жолдар болса.

Бет өлшемі, іздеу немесе сүзгілер өзгергенде есептегіш бірінші бетке қайта орнатылады.

### Үлкен журналдар (екі деңгейлі жүктеу)

Өте көп деректері бар беттерде **екі деңгей** қолданылады:

| Бөлім | API жүктеу | Кестеде беттеу |
|-------|------------|----------------|
| **MQTT** (`/mqtt`) | Кесте үстіндегі **«Тағы жүктеу (100 жазба)»** — серверден телеметрияның келесі беті | DataTable: бетте 20–100 жол, Артқа/Алға, Тағы жүктеу |
| **Карталар** | Солай, API-дан **100** пакеттер | DataTable төменгі бөлігінде |
| **Пост күй тарихы** (`/posts/:id`) | `state/process` телеметриясы күн сүзгісімен порциялармен жүктеледі | DataTable: мыңдаған жолды бірден көрсетпейді |

Шолудағы **«Соңғы хабарландырулар»** виджеті — ықшам алдын ала қарау (әдепкі 10 жол, **10** мәні «Бетте» тізімінде қолжетімді).

## Бөлімдер толығырақ

### Шолу (`/`)

Инкассацияға дейінгі KPI карточкалары: қолма-қол / қолма-қолсыз / жалпы кіріс / жеңілдік сомасы / белсенді қателер.

Қаржы және пайдалану сомалары — **әр посттың соңғы жазбаларының қосындысы** (барлық тарихи жолдар емес).

Графиктер (Recharts): пост күйі, **дөңгелек диаграммалар** «Пайдалану» (клиенттер / сервис / VIP) және төлем үлестері (қолма-қол / қолма-қолсыз / жеңілдіктер), күн бойынша кіріс. Соңғы хабарландырулар кестесі.

### Автомойкалар (`/washes`)

Объектілер анықтамалығы: атауы, сипаттамасы, мекенжайы, жасалған күні, бұлт. «Посттар саны» бағаны.

### Посттар (`/posts`)

Автомойкаға байланысты посттар: **онлайн/оффлайн күйі**, нөмір, атауы, **сериялық нөмір**, **MQTT-логин**, объект. Байланысты деректерді каскадты жою.

Әрекеттер бағанында **⚙** белгішесі (құрылғы баптаулары) → пост беті. Жасау/редакциялау кезінде — пост панелі үшін MQTT логин мен пароль.

**Онлайн** — `lastMessageAt` 30 секундтан ескі емес болса (Шолудағыдай).

### Пост карточкасы (`/posts/:id`)

| Блок | Сипаттама |
|------|----------|
| Тақырып | **Онлайн/оффлайн күйі**, атауы, сериялық нөмір |
| Пост сипаттамасы | Атауы, объект мекенжайы, сериялық нөмір, ТО, мүмкіндіктер |
| **Құрылғы баптаулары** | Режим бағалары, MQTT командалары, `dt_pref` префиксі — қараңыз [MQTT](mqtt.md) |
| Күй тарихы | Күн сүзгісімен `state/process` телеметриясы кестесі; **20–100** жол беттеу, **Тағы жүктеу** |

`#device-settings` якорь құрылғы баптаулары блокына айналдырады.

### Ағымдағы күй (`/states`)

Барлық объектілердің барлық посттары. **Күй** бағаны (онлайн/оффлайн), күй бойынша сүзгі.

Толық ендік **интерактивті график** (Recharts): метрикаларды ауыстыру (режим уақыты / баланс / тегін пауза), tooltip, көп пост болса brush, режим уақыты live-жаңарту.

Кесте: мекенжай, пост нөмірі, баланс, пауза, жеңілдік, режим live-таймері, режим атауы, соңғы хабарлама күні.

### Карталар (`/cards`)

| Қосалқы бөлім | `cardType` |
|---------------|------------|
| Жеңілдік | `regular` |
| Сервистік | `service` |
| VIP | `unlimited` |

**Қолдану журналы:** әр NFC оқиғасы (`state/card` посттан) жаңа жол жасайды. Белсенді сессия `state/process` деректерімен жаңартылады (баланс, жеңілдік).

Жеңілдік түрі — анықтамалықтан `1`–`5` нөмірі. Күйлер: `success`, `rejected`. Инкассация (`cardType: collection` құрылғыда) — CRM-де хабарландыру, карталар бөлімінде жол жоқ.

### Аналитика (`/usage`, `/finance`, `/archive`)

- **Пайдалану** — инкассациядан бұрын/кейін, regular/service/unlimited категориялары.
- **Қаржы** — қолма-қол, қолма-қолсыз, жеңілдік, кіріс.
- **Архивтеу** — деректер топтары бойынша саясаттар, операциялар журналы.

### Жүйе және автоматтандыру

| Бөлім | Маршрут | Сипаттама |
|-------|---------|----------|
| Хабарландырулар | `/notifications` | CRM-хабарландырулар, оқылған белгілеу; web/Telegram арналары баптаулары **Шолуда** |
| **Профиль** | `/profile` | Аты, email, пароль ауыстыру *(тақырыптағы сілтеме)* |
| **Баптау шебері** | `/setup` | CRM бастапқы баптауы; қайта іске қосу `/setup?restart=1` *(Admin/Operator)* |
| **Пайдаланушылар** | `/users` | Dynamic API есептері, **Telegram user_id**, топ тағайындау *(Admin)* |
| **Топтар мен құқықтар** | `/groups` | RBAC топтары және permissions матрицасы *(Admin)* |
| **Баптаулар** | `/settings` | MQTT, Telegram notify, CRM жаңарту, әдепкі валюта |
| Логтар | `/logs` | Dynamic API аудиті *(Admin)* |
| **Ақпарат** | `/info-messages` | Ақпараттық бот үшін жаңалықтар мен акциялар; «Кесте бойынша» күйі → уақыт келгеннен кейін **«Жарияланды»** *(Admin)* |
| Telegram | `/telegram` | PyOrchestrator боттары: **Басқару** / **Сервистік** / **Ақпараттық**; QR-сілтеме; жаппай әрекеттер; v3.1 / **v1.9** үлгілері *(Admin, PyOrch)* |

| Резервтік көшірмелер | `/backups` | MongoDB backup, қолмен іске қосу, жүктеу *(Admin)* |
| Жұмыс режимдері | `/work-modes` | 0–9 режимдер анықтамалығы *(Admin)* |
| Валюталар | `/currency` | `/api/crm/currencies` анықтамалығы *(Admin)* |
| Жеңілдік түрлері | `/discount-types` | 1–5 нөмірлері мен атаулары *(Admin)* |

### Навигация тұрақтылығы (v1.1.12)

- желі сбойында JS-чанктарды қайта жүктеу (`lazyPage`);
- сұр экран орнына `RouteErrorBoundary`;
- polling таймауты 60 с; Шолуда — шексіз Loading орнына қате хабарламасы.

### Кіру беті (`/login`)

Екі бағанды layout (PyOrchestrator сияқты): сол жақта брендинг SVG-фонмен (толқындар/SCADA), оң жақта форма. Оң жоғарғы бұрышта тақырып ауыстырғышы. JWT авто-жаңарту; сессия аяқталса — `/login`-ге бағыттау.
