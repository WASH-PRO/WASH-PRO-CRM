v1.1.43 — PyOrch stability & faster Modules page · Стабильность PyOrch и быстрее «Модули»

---

## English

### Fixed

- **PyOrchestrator** — commit run to Postgres before Redis enqueue (fixes race: runtime `/internal/runs/start` → 500 `NoResultFound`); purge stale `runtime:jobs` entries on script stop
- **modules-bridge** — 8s timeouts for PyOrch and CRM profile checks; parallel run-status fetch in catalog; 10s status cache; recover stops module before restart; auto-retry start when `Max concurrent runs reached`
- **Dashboard `/modules`** — load health and catalog in parallel (less spinner time)

### Changed

- **crm-update-build** — rebuilds `pyorch-backend` on CRM auto-update when `PYORCHESTRATOR_ENABLED=true`
- **dynamic-pricing catalog v1.1.6** — poll interval min 5 sec, step 5 sec

### Upgrade

```bash
git pull
# Or: Settings → Software updates → Update

docker compose -f docker-compose.yml -f docker-compose.pyorchestrator.yml build pyorch-backend modules-bridge dashboard
docker compose -f docker-compose.yml -f docker-compose.pyorchestrator.yml up -d pyorch-backend modules-bridge dashboard
```

If modules were stuck: **Automation → Modules** — Stop → Start for each running module.

## Русский

### Исправлено

- **PyOrchestrator** — commit run в PostgreSQL до enqueue в Redis (гонка `/internal/runs/start` → 500); очистка `runtime:jobs` при остановке скрипта
- **modules-bridge** — таймауты PyOrch и CRM 8 с; параллельный статус в каталоге; кэш 10 с; recover: stop перед start; повтор start при `Max concurrent runs reached`
- **Страница `/modules`** — параллельная загрузка health + catalog

### Изменено

- **Автообновление CRM** — пересборка `pyorch-backend` при `PYORCHESTRATOR_ENABLED=true`
- **Каталог dynamic-pricing v1.1.6** — интервал от 5 сек, шаг 5 сек

### Обновление

```bash
git pull
# Или: Настройки → Обновления ПО → Обновить

docker compose -f docker-compose.yml -f docker-compose.pyorchestrator.yml build pyorch-backend modules-bridge dashboard
docker compose -f docker-compose.yml -f docker-compose.pyorchestrator.yml up -d pyorch-backend modules-bridge dashboard
```

Если модули зависли: **Автоматизация → Модули** — Стоп → Запуск для каждого модуля.
