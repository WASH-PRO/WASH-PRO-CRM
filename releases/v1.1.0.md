## 🇬🇧 English

Major release after 1.0.0: MQTT migration, setup wizard, Telegram bot v2.7 with CRM auth, updated Dashboard.

### Telegram bots (template v2.7)

- Unified reports: status, washes, posts, revenue, statistics, cards
- Post modes from **Work modes** reference (`program_N` → human-readable names)
- **CRM auth:** **Telegram user_id** on users + RBAC group
- Viewer — read-only; Operator — create & commands; Administrator — full access
- Strangers see "Private bot" only
- Fixed duplicate replies: token lock, legacy bot shutdown, send deduplication

### Dashboard

- Overview pie charts: **Usage** (clients / service / VIP) and payment mix
- Telegram user_id on user form
- Bot list no longer disappears on start/stop

### MQTT & telemetry

- **RabbitMQ → Mosquitto**; migration: `./scripts/migrate-to-mqtt.sh`
- Post isolation by serial in ACL; MQTT accounts on post card
- Device settings: mode prices, commands, `dt_pref`
- DLQ journal; finance/usage stats upsert

### Other

- Setup wizard (`/setup`), post online/offline (30 s threshold)
- User profile, load-more pagination, cron auto-archive
- PyOrchestrator vendored **v0.1.13**

### Upgrade

```bash
git pull
# set APP_VERSION=1.1.0 in .env if needed
docker compose -f docker-compose.yml -f docker-compose.pyorchestrator.yml up -d --build
```

## 🇷🇺 Русский

Крупный релиз после 1.0.0: переход на MQTT, мастер настройки, Telegram-бот v2.7 с авторизацией через CRM.

### Telegram-боты (шаблон v2.7)

- Единый формат отчётов: статус, автомойки, посты, выручка, статистика, карты
- Режимы из справочника «Режимы работы»
- **Авторизация по CRM:** **Telegram user_id** + группа RBAC
- Viewer — только просмотр; Operator — создание и команды; Administrator — полный доступ
- Посторонним — «Частный бот»
- Исправлены дубли ответов: lock по токену, остановка legacy-ботов

### Dashboard

- Круговые диаграммы на Обзоре: **Использование** и доли оплаты
- Поле Telegram user_id в форме пользователя
- Список ботов не пропадает при старт/стоп

### MQTT и телеметрия

- **RabbitMQ → Mosquitto**; миграция: `./scripts/migrate-to-mqtt.sh`
- Изоляция постов по serial; учётные записи MQTT в карточке поста
- Настройки устройства: цены, команды, `dt_pref`
- DLQ-журнал; upsert статистики

### Прочее

- Мастер настройки (`/setup`), онлайн/оффлайн постов (30 с)
- Профиль, пагинация «Загрузить ещё», автоархив по cron
- PyOrchestrator **v0.1.13**

### Обновление

```bash
git pull
# APP_VERSION=1.1.0 в .env при необходимости
docker compose -f docker-compose.yml -f docker-compose.pyorchestrator.yml up -d --build
```
