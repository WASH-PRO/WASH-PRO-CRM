v1.1.39 — Fix CRM auto-update build step · Исправление шага «Сборка» при автообновлении

---

## English

### Fixed

- **CRM auto-update failed on Build** — `compose-files.sh` used bash-only `${BASH_SOURCE…}` while update-bridge ran steps via `sh`; executor now uses **bash**, `SCRIPT_DIR` resolves from `DEPLOY_ROOT`

### Upgrade

1. **Settings → Software updates** → update to **v1.1.39** (Build step should complete)
2. If v1.1.38 update failed: click **Update** again after pulling this release

## Русский

### Исправлено

- **Автообновление CRM падало на «Сборка»** — `compose-files.sh` использовал bash-синтаксис `${BASH_SOURCE…}`, а update-bridge запускал шаги через `sh`; executor теперь через **bash**, `SCRIPT_DIR` из `DEPLOY_ROOT`

### Обновление

1. **Настройки → Обновления ПО** → **v1.1.39** (шаг «Сборка» должен завершиться)
2. Если v1.1.38 завершился с failed — нажмите **Обновить** снова после этого релиза
