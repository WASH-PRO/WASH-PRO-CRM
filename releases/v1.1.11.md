## 🇬🇧 English

### Added

- **Information bot v1.8** — news/promotions feed via menu buttons; auto-push to subscribers; images sent as **files** (not URL)
- **Private chat isolation** — all bot types work **only in private chats**; in groups the bot refuses and asks to open a DM

### Fixed

- **"News (N)" without text** — reliable text/photo delivery with fallback; service JWT when public API is unavailable
- **"No news"** — expired `expiresAt` filter; default **Published** status; auto `publishedAt`
- **Auto-broadcast** — timestamp baseline; news without manual publish date
- **Management bot v3.1** — private chats only; "Private bot" hint with isolation explanation

### Changed

- **Information** section — hint for **Hide after** field; ignore end date ≤ publish date

### Upgrade

```bash
git pull
docker compose -f docker-compose.yml -f docker-compose.pyorchestrator.yml up -d --build pyorch-bridge dashboard init-seed
# Dashboard → Telegram → ▶ on the information bot
```

## 🇷🇺 Русский

### Добавлено

- **Информационный бот v1.8** — лента новостей/акций по кнопкам; автоматическая рассылка подписчикам; изображения **файлом** (не по URL)
- **Изоляция личных чатов** — все типы ботов только в **личных сообщениях**; в группах бот не отвечает и подсказывает открыть личный чат

### Исправлено

- **«Новости (N)» без текста** — надёжная отправка текста и фото с fallback
- **«Нет новостей»** — фильтр просроченного `expiresAt`; статус «Опубликовано» по умолчанию
- **Авторассылка** — baseline по timestamp
- **Управляющий бот v3.1** — только личные чаты

### Изменено

- **Раздел «Информация»** — подсказки по полю «Скрыть после»

### Обновление

```bash
git pull
docker compose -f docker-compose.yml -f docker-compose.pyorchestrator.yml up -d --build pyorch-bridge dashboard init-seed
# Dashboard → Telegram → ▶ у информационного бота
```
