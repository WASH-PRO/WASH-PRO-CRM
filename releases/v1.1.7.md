## 🇬🇧 English

### Fixed

- **In-app updater** — when running from `update-bridge` container, Mosquitto bind mounts and build context use the real host path (`WASH_HOST_PROJECT_ROOT`); build step no longer restarts Mosquitto; rebuilds `dynamic-api`, `dynamic-api-panel`, `init-seed`; writes `APP_VERSION` to `.env` after `git pull`

### Upgrade

If updates from Settings still fail (old `update-bridge`), run once manually:

```bash
git pull
docker compose up -d --build update-bridge
```

Then use **Settings → Software updates** in Dashboard.

## 🇷🇺 Русский

### Исправлено

- **Автообновление из настроек** — bind mount'ы Mosquitto и build context используют реальный путь хоста (`WASH_HOST_PROJECT_ROOT`); шаг сборки не перезапускает Mosquitto; пересобираются `dynamic-api`, `dynamic-api-panel`, `init-seed`; после `git pull` в `.env` записывается `APP_VERSION`

### Обновление

Если обновление из настроек ещё не работает:

```bash
git pull
docker compose up -d --build update-bridge
```

После этого обновления из CRM → Настройки должны проходить штатно.
