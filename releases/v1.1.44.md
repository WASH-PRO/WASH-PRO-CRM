---

## English

### Added

- **White-label branding** — Settings → **Branding**: product name, tagline, logo URL, support and docs links; shown on login, welcome, sidebar, and System page
- **Toast notifications** and **confirm modal** — replace browser `alert`/`confirm` on Settings, Users, Groups, Backups, integrity repair, and software updates
- **Support diagnostics** — System → **Information**: download JSON report (versions, health, backups, settings keys)
- **Full backup bundle** — Settings → Backups: optional `fullBundle` adds `*-extras.tar.gz` with CRM settings export and `modules/installed/*/data/` alongside MongoDB archive
- **In-app help** — sections **Setup wizard**, **Welcome**, **Profile**; link to full docs (GitHub Pages) in help sidebar
- **CI** — GitHub Actions: sync version docs, validate EN/RU i18n keys, docs links, dashboard build
- **`scripts/validate-i18n-keys.sh`** — parity check for core and help messages

### Changed

- **Settings i18n** — localized section titles (PyOrchestrator, Dynamic API, MQTT); branding and full-bundle labels
- **init-seed** — default `branding` and `backup.fullBundle: true` for new installs
- **wash-backup** — read-only mount `modules:/modules`; cleanup removes paired `-extras.tar.gz` with Mongo archive
- **Tailwind** — `panel-surface` / `panel-bg` tokens for cards and modals
- **Module catalog** — EN/RU README and help URLs (see prior commits `77725c1`, `1e0e6f3`)

### Upgrade

```bash
git pull
# Or: Settings → Software updates → Update

docker compose build dashboard backup init-seed
docker compose up -d dashboard backup
```

After upgrade, open **Settings → Branding** to customize product name (optional).  
**MongoDB:** default compose image is `mongo:4.4` (CPUs without AVX). For existing **mongo:7** data, keep `docker-compose.override.yml` with `image: mongo:7` — see `docker-compose.override.yml.example`.

## Русский

### Добавлено

- **White-label брендинг** — Настройки → **Брендинг**: название продукта, слоган, URL логотипа, ссылки поддержки и документации; на экране входа, приветствии, в сайдбаре и на странице «Информация»
- **Toast-уведомления** и **модальное подтверждение** — вместо `alert`/`confirm` в Настройках, Пользователях, Группах, Резервных копиях, целостности и обновлениях ПО
- **Диагностика для поддержки** — Система → **Информация**: скачивание JSON-отчёта (версии, health, бэкапы, ключи настроек)
- **Полный пакет бэкапа** — Настройки → Резервное копирование: опция `fullBundle` добавляет `*-extras.tar.gz` с экспортом настроек CRM и `modules/installed/*/data/` вместе с архивом MongoDB
- **Встроенная справка** — разделы **Мастер настройки**, **Приветствие**, **Профиль**; ссылка на полную документацию (GitHub Pages) в сайдбаре справки
- **CI** — GitHub Actions: синхронизация версии в docs, проверка паритета EN/RU i18n, ссылок в docs, сборка dashboard
- **`scripts/validate-i18n-keys.sh`** — проверка паритета ключей core и help

### Изменено

- **i18n Настроек** — локализованные заголовки секций (PyOrchestrator, Dynamic API, MQTT); подписи брендинга и полного бэкапа
- **init-seed** — дефолты `branding` и `backup.fullBundle: true` для новых установок
- **wash-backup** — read-only mount `modules:/modules`; очистка старых копий удаляет парный `-extras.tar.gz`
- **Tailwind** — токены `panel-surface` / `panel-bg` для карточек и модалок
- **Каталог модулей** — README и help EN/RU (коммиты `77725c1`, `1e0e6f3`)

### Обновление

```bash
git pull
# Или: Настройки → Обновления ПО → Обновить

docker compose build dashboard backup init-seed
docker compose up -d dashboard backup
```

После обновления при необходимости откройте **Настройки → Брендинг**.  
**MongoDB:** в compose по умолчанию `mongo:4.4` (CPU без AVX). Для данных на **mongo:7** сохраните `docker-compose.override.yml` с `image: mongo:7` — см. `docker-compose.override.yml.example`.
