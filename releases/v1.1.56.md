v1.1.56 — Unlimited runtime for bots and daemon modules · Безлимит для ботов и daemon-модулей

---

## English

### Fixed

- **PyOrchestrator wall timeout** — bots and daemon modules no longer die after 1h (default) / 24h. `max_runtime_seconds: 0` means unlimited wall clock and CPU rlimit.
- **modules-bridge / pyorch-bridge** — set unlimited runtime on create/start so existing scripts pick it up on next Start.

### Upgrade

```bash
# Dashboard → Settings → Software updates → CRM
# then Modules → Washes Nearby → Start
# and restart Telegram bots if needed
```

---

## Русский

### Исправлено

- **Таймаут PyOrchestrator** — боты и daemon-модули больше не падают через 1 ч / 24 ч. `max_runtime_seconds: 0` = безлимит по wall clock и CPU.
- **modules-bridge / pyorch-bridge** — при создании/старте выставляют безлимит (старые скрипты подхватят при следующем Start).

### Обновление

```bash
# Настройки → Обновления ПО → CRM
# затем Модули → Автомойки рядом → Старт
# и перезапуск Telegram-ботов при необходимости
```
