## English

### Fixed

- **Post detail** — single «Load more» button for server-side history pagination (removed duplicate DataTable button)
- **Archive page** — result status column, clearer zero-record display, manual telemetry purge button, scanned count in log entries

### Upgrade

Auto-update from Dashboard (**Settings → Updates**) or:

```bash
git pull
docker compose build dashboard backup
docker compose up -d dashboard backup
```

Standard CRM auto-update (pull → build → seed → health) applies this release safely; no database migrations required.

## Русский

### Исправлено

- **Страница поста** — одна кнопка «Загрузить ещё» для серверной догрузки истории (убран дубль из DataTable)
- **Архивирование** — столбец «Результат», понятное отображение нуля записей, ручной запуск очистки телеметрии, счётчик проверенных записей в журнале

### Обновление

Автообновление из Dashboard (**Настройки → Обновления**) или:

```bash
git pull
docker compose build dashboard backup
docker compose up -d dashboard backup
```

Стандартное автообновление CRM (pull → build → seed → health) применяет релиз безопасно; миграции БД не требуются.
