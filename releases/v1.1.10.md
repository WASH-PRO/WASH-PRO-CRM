## 🇬🇧 English

### Added

- **QR code and bot link** — button in Telegram bots table; modal with `t.me` link and QR
- **Bulk actions** on Telegram bots page — CSV export, start, stop, delete selected
- **Information bot v1.3** — background news/promotions push; last 10 entries via button; `@username` registration on start

### Changed

- **Public information bot access** — no Telegram ID check; public CRM endpoints for feed and references

### Fixed

- **Bot link unavailable / fetch failed** — bridge internet access; `TELEGRAM_TOKEN` from PyOrchestrator; username registered on bot start; clearer errors

### Upgrade

```bash
git pull
docker compose -f docker-compose.yml -f docker-compose.pyorchestrator.yml up -d --build pyorch-backend pyorch-bridge dashboard init-seed
```

Restart Telegram bots (bulk **Start**), then open QR code.

## 🇷🇺 Русский

### Добавлено

- **QR-код и ссылка на бота** — кнопка в таблице Telegram-ботов
- **Массовые действия** — экспорт CSV, запуск, остановка, удаление выбранных
- **Информационный бот v1.3** — рассылка новостей/акций, лента по кнопкам, регистрация `@username` при старте

### Изменено

- **Публичный доступ** информационного бота — без Telegram ID; публичные CRM endpoints

### Исправлено

- **Ссылка на бота / fetch failed** — доступ bridge к Telegram API, токен из PyOrchestrator, username при запуске

### Обновление

```bash
git pull
docker compose -f docker-compose.yml -f docker-compose.pyorchestrator.yml up -d --build pyorch-backend pyorch-bridge dashboard init-seed
```

Перезапустите ботов (массовое «Запустить»), затем откройте QR-код.
