v1.1.23 — Update job fixes · Исправление автообновления

**English** · **Русский**

---

## 🇬🇧 English

### Fixed

- **False CRM version after failed update** — `APP_VERSION` in `.env` is written only after a **successful** job; reverted on failure
- **Running version detection** — CRM current version reads from live dashboard `version.json`, not stale `.env`
- **Stuck failed job on card** — **Retry** and **Hide error** buttons; dismissed failed jobs no longer block the UI
- **Misleading update state** when build failed but pull had already bumped `.env`

### Changed

- Dashboard Docker image exposes `/version.json` with baked `VITE_APP_VERSION`
- Clearer failed-update hint (Docker Hub timeout, disk space)

### Upgrade

```bash
git pull
docker compose up -d --build update-bridge dashboard
```

Set `APP_VERSION=1.1.23` in `.env` only **after** a successful update, or let the updater set it automatically.

Docs: https://wash-pro.github.io/WASH-PRO-CRM/en/

## 🇷🇺 Русский

### Исправлено

- **Ложная версия CRM после ошибки обновления** — `APP_VERSION` в `.env` пишется только после **успешного** job; при ошибке откатывается
- **Определение работающей версии** — текущая версия CRM читается из `version.json` работающего dashboard, а не из устаревшего `.env`
- **Зависшая ошибка на карточке** — кнопки **«Повторить»** и **«Скрыть ошибку»**; скрытые failed job больше не блокируют UI
- **Неверное состояние обновления**, когда сборка упала, а pull уже успел изменить `.env`

### Изменено

- Образ dashboard отдаёт `/version.json` с версией сборки
- Уточнена подсказка при ошибке (таймаут Docker Hub, место на диске)

### Обновление

```bash
git pull
docker compose up -d --build update-bridge dashboard
```

`APP_VERSION=1.1.23` в `.env` — только **после** успешного обновления или автоматически через updater.

Документация: https://wash-pro.github.io/WASH-PRO-CRM/ru/
