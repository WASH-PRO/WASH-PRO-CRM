v1.1.21 — Git safe.directory fix · Исправление проверки git в /deploy

**English** · **Русский**

---

## 🇬🇧 English

### Fixed

- **False “/deploy is not a git repository”** — integrity check now registers `git safe.directory /deploy` before `git rev-parse`; fixes **dubious ownership** when `.git` is owned by a different user than the container (common on production)
- **Startup** — `update-bridge` auto-registers `/deploy` as a safe directory on launch
- **Stale failed job on card** — after a successful update, old failed jobs no longer stay on the component card (only the latest job for that component)
- **Generic exit code 1** — shell errors now include the last lines of command output for easier diagnosis

### Upgrade

```bash
git pull
docker compose up -d --build update-bridge dashboard
```

Set `APP_VERSION=1.1.21` in `.env` if needed.

> **`/deploy`** inside `update-bridge` is a bind mount of your project on the host — it **should** be a git repo. If you see a git warning, apply **git safe.directory** in Integrity repair or upgrade to v1.1.21.

Docs: https://wash-pro.github.io/WASH-PRO-CRM/en/

## 🇷🇺 Русский

### Исправлено

- **Ложное «Каталог /deploy не является git-репозиторием»** — проверка целостности регистрирует `git safe.directory /deploy` перед `git rev-parse`; устраняет **dubious ownership**, когда `.git` принадлежит другому пользователю (типично на production)
- **Старт** — `update-bridge` автоматически добавляет `/deploy` в safe.directory при запуске
- **Старый failed job на карточке** — после успешного обновления прежние ошибки больше не висят на карточке (только последний job компонента)
- **«Команда завершилась с кодом 1»** — в текст ошибки добавлены последние строки вывода команды

### Обновление

```bash
git pull
docker compose up -d --build update-bridge dashboard
```

При необходимости: `APP_VERSION=1.1.21` в `.env`.

> **`/deploy`** внутри `update-bridge` — bind mount проекта на хосте; **должен** быть git-репозиторием. При предупреждении git — **git safe.directory** в «Целостность» или обновитесь до v1.1.21.

Документация: https://wash-pro.github.io/WASH-PRO-CRM/ru/
