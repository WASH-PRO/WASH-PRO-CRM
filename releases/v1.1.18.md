v1.1.18 — Software updates fix · Исправление системы обновлений

**English** · **Русский**

---

## 🇬🇧 English

### Fixed

- **Update check without GitHub token** — `update-bridge` falls back to `git ls-remote` when GitHub REST API rate limit is hit (60 req/h per IP without token); public installs no longer lose update notifications
- **Notification reset on page load** — Dashboard no longer forces GitHub refresh on F5 or during update progress polling; only **Check now** hits GitHub
- **Stale cache on API errors** — last known release versions are kept when a check fails instead of wiping `latestVersion`
- **`git pull` blocked by local edits** — documented pattern: server-specific settings in untracked `docker-compose.override.yml` and `local/apply-server-patches.sh` (see deployment docs)

### Changed

- **Software updates polling** — cached status for normal load and 3 s job progress; explicit `checkGithub` for manual refresh
- **Examples** — `docker-compose.override.yml.example`, `local/apply-server-patches.sh.example` for CPU without AVX (MongoDB 4.4) and post-update patches

### Upgrade

```bash
git pull
docker compose up -d --build update-bridge dashboard
```

Set `APP_VERSION=1.1.18` in `.env` if the updater does not bump it automatically.

> **Public installs:** `GITHUB_TOKEN` is **optional**. Without it, version checks use Git; release notes in UI may be empty until you click **Check now** with API quota available.

Docs: https://wash-pro.github.io/WASH-PRO-CRM/en/

## 🇷🇺 Русский

### Исправлено

- **Проверка обновлений без GitHub-токена** — `update-bridge` при лимите GitHub REST API (60 запросов/ч с IP без токена) переключается на `git ls-remote`; публичные установки снова видят доступные версии
- **Сброс уведомления при загрузке страницы** — Dashboard больше не дергает GitHub при F5 и при опросе прогресса обновления; принудительная проверка — только кнопка **«Проверить сейчас»**
- **Потеря версий при ошибке API** — при сбое проверки сохраняются последние известные `latestVersion`, а не обнуляются
- **Блокировка `git pull` локальными правками** — описан паттерн: серверные настройки в неотслеживаемых `docker-compose.override.yml` и `local/apply-server-patches.sh` (см. развёртывание)

### Изменено

- **Опрос обновлений** — обычная загрузка и прогресс job (каждые 3 с) читают кэш; GitHub — только по **«Проверить сейчас»**
- **Примеры** — `docker-compose.override.yml.example`, `local/apply-server-patches.sh.example` (MongoDB 4.4 без AVX, патчи после pull)

### Обновление

```bash
git pull
docker compose up -d --build update-bridge dashboard
```

Укажите `APP_VERSION=1.1.18` в `.env`, если апдейтер не обновил версию автоматически.

> **Публичные установки:** `GITHUB_TOKEN` **не обязателен**. Без него версии берутся через Git; тексты release notes в UI могут быть пустыми, пока не сработает API (кнопка «Проверить сейчас»).

Документация: https://wash-pro.github.io/WASH-PRO-CRM/ru/
