v1.1.50 — Fix stuck update spinner · Спиннер обновления

---

## English

### Fixed

- **Software updates** — banner no longer spins forever on a stuck `queued` job when the target version is already installed
- **update-bridge** — auto-closes stale queued jobs; queued updates run after the current job finishes instead of being dropped

### Upgrade

Auto-update from Dashboard (**Settings → Updates**) or:

```bash
git pull
docker compose build dashboard update-bridge
docker compose up -d dashboard update-bridge
```

After CRM auto-update, restart update-bridge once if the spinner persists: `docker compose restart update-bridge`

## Русский

### Исправлено

- **Обновления ПО** — баннер больше не крутит спиннер на зависшей задаче `queued`, если целевая версия уже установлена
- **update-bridge** — автозакрытие зависших задач; следующее обновление в очереди запускается после текущего

### Обновление

Автообновление из Dashboard (**Настройки → Обновления**) или:

```bash
git pull
docker compose build dashboard update-bridge
docker compose up -d dashboard update-bridge
```

После автообновления CRM при необходимости перезапустите update-bridge: `docker compose restart update-bridge`
