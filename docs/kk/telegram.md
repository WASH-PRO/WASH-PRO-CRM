WASH PRO CRM Telegram-боттары **PyOrchestrator** және **`pyorch-bridge`** сервисі арқылы жұмыс істейді. Басқару — **Dashboard → Автоматтандыру → Telegram** ішінде (Administrator рөлі қажет).

## Сұлба

```
Telegram пайдаланушысы (жеке чат)
        ↓
   Telegram API (long polling)
        ↓
PyOrchestrator runtime (botTemplate / infoBotTemplate ішіндегі main.py)
        ↓
Dynamic API  —  CRM endpoints (JWT service немесе public)
```

| Компонент | Мақсаты |
|-----------|---------|
| `pyorch-bridge` | Боттар жасау, secrets, үлгі синхрондауы, restart, QR/сілтеме |
| `botTemplate.ts` | Басқару және сервистік бот (үлгі **v3.1**) |
| `infoBotTemplate.ts` | Ақпараттық бот (үлгі **v1.9**) |
| Dynamic API | RBAC, `/api/crm/info-messages` лентасы, `/api/users/telegram/{id}/auth` авторизациясы |

## Бот түрлері

| Түр | Мақсаты | Қолжетімділік |
|-----|---------|---------------|
| **Басқару** | Толық операторлық бот: мониторинг, анықтамалықтар, пост командалары | Telegram ID бойынша CRM RBAC |
| **Сервистік** | Объектілерді жасау/жоюсыз мониторинг және пост командалары | CRM RBAC |
| **Ақпараттық** | Жария мәзір: жаңалықтар, бағалар, бос/бос емес, акциялар | **Telegram ID жоқ** — жеке чатта кез келген пайдаланушы |

Бот жасау кезінде түрін таңдаңыз — командалар пресеті автоматты қойылады.

## Жеке чаттар және оқшаулау (v1.1.11)

Барлық боттар **тек жеке хабарламаларда** жұмыс істейді:

- әр пайдаланушы ботпен **тек өз** диалогын көреді;
- жауаптар, мәзірлер және тарату топтық чаттарға түспейді;
- бот топқа қосылса — онда жауап бермейді және жеке чатқа кеңес жібереді: QR/сілтеме арқылы ботты ашып `/start` басыңыз.

Боттар кестесіндегі **QR-код және сілтеме** (`t.me/...`) пайдаланушыны **жеке чатқа** жетелейді.

## Ақпараттық бот

### Мәзір

| Батырма | Мазмұны |
|---------|---------|
| 📰 Жаңалықтар | Соңғы 10 жарияланған жаңалық |
| 💰 Бағалар | Таңдалған мойка бойынша режимдер мен бағалар |
| 🅿️ Бос/бос емес | Бос және бос емес посттар |
| 🎁 Акциялар | Соңғы 10 акция |

### Мазмұн — «Ақпарат» бөлімі

**Dashboard → Автоматтандыру → Ақпарат** (`/info-messages`):

| Өріс | Ұсыныс |
|------|--------|
| **Күй** | «Жарияланды» — әйтпесе бот жазбаны көрсетпейді. «Кесте бойынша» өткен күнмен кестеде **Жарияланды** ретінде көрсетіледі (v1.1.12) |
| **Жариялау күні** | Міндетті емес — жариялау кезінде автоматты қойылады |
| **Кейін жасыру** | Жаңалық жойылмауы керек болса **бос қалдырыңыз**. Жариялау күнінен ≤ күн қоймаңыз |
| **Сурет URL** | jpg/png/webp тікелей сілтемесі 10 МБ-қа дейін; бот файлды жүктеп Telegram-ға жібереді |

### Автотарату

1. Пайдаланушы ботқа **жеке чатта** `/start` жібереді (жазылушы тіркелуі).
2. CRM-де жаңалық жарияланғанда бот жазылушыларға таратады (~30 с).
3. Пайдаланушының алғашқы `/start`-інен **бұрын** жарияланған жаңалықтар автотаратуға кірмейді — оларды **📰 Жаңалықтар** батырмасымен ашуға болады.

### Суреттер (v1.9)

Бот `imageUrl` бойынша суретті **жүктейді** және **бір хабарламамен** жібереді (фото + жазба, мәтін дубликаты жоқ). Фото қолжетімсіз болса — жаңалық мәтіні келеді. URL талаптары:

- интернеттен қолжетімді (`pyorch-runtime` контейнері);
- файлына тікелей сілтеме;
- өлшемі 10 МБ-қа дейін.

## Басқару және сервистік бот

### Жасау

1. `.env` ішінде `PYORCHESTRATOR_ENABLED=true`, содан кейін `./scripts/start.sh`
2. **Dashboard → Telegram → Бот жасау**
3. [@BotFather](https://t.me/BotFather) токені, түр, командалар
4. Іске қосу (▶)

```bash
docker compose -f docker-compose.yml -f docker-compose.pyorchestrator.yml up -d --build pyorch-bridge
```

Барлық боттарда үлгіні мәжбүрлі жаңарту: `POST /api/telegram-bots/bots/refresh`.

### Қызметкерлер авторизациясы

1. Telegram ID — [@userinfobot](https://t.me/userinfobot)
2. **Dashboard → Пайдаланушылар** → **Telegram user_id** өрісі
3. RBAC тобы (Viewer / Operator / Administrator)

Бот `GET /api/users/telegram/{telegramUserId}/auth` шақырады. Бөгде адамдарға — тек өз ID-імен «Жеке бот» хабарламасы.

| Permission | Ботта |
|------------|-------|
| `view` | Мониторинг, есептер |
| `create` | Мойкалар мен посттар жасау |
| `update` | Пост командалары |
| `delete` | Объектілерді жою |

### Жаппай әрекеттер (v1.1.10)

**Telegram** бетінде — жол чекбокстары: CSV экспорт, іске қосу, тоқтату, таңдалған боттарды жою.

## Сенімділік

| Механизм | Сипаттама |
|----------|----------|
| Токен бойынша lock | Бір токенге бір polling процесі |
| Offset / state | Restart кейін дубликат жоқ |
| Тек private | Топтар мен арналар еленбейді |
| Үлгі | bridge іске қосылғанда және restart алдында `syncBotCode` |
| Username | Бот іске қосылғанда `@username` тіркелуі + QR үшін PyOrchestrator-дан токен оқу |

## Ақаулықтарды жою

| Белгі | Әрекет |
|-------|--------|
| «PyOrchestrator қолжетімсіз» | `PYORCHESTRATOR_ENABLED=true`, health `curl localhost/api/telegram-bots/health` |
| «Жаңалық жоқ» / бос лента | Күй **Жарияланды**; «Кейін жасыру» тазалаңыз; ботты restart |
| «Жаңалықтар (2)» мәтін/фото жоқ | `pyorch-bridge` жаңартыңыз, ботты restart (v1.9+); сурет URL тексеріңіз |
| Автотарату жоқ | Пайдаланушы жеке чатта `/start` басуы керек; **Жарияланды** күйі; ~30 с күтіңіз |
| Бот сілтемесі қолжетімсіз | Ботты restart; қажет болса бот баптауларында токенді сақтаңыз |
| Бот топта жауап береді | Күтілетін — өшірілген; QR арқылы жеке чат |
| Қызметкерге «Жеке бот» | **Пайдаланушылар** → Telegram user_id, active күй |
| Бот үнсіз | `./scripts/fix-pyorch.sh`, логтар `docker logs wash-pyorch-runtime` |

Толығырақ: [Ақаулықтарды жою](troubleshooting.md).

## Bridge API (admin JWT)

| Әдіс | Жол | Сипаттама |
|------|-----|----------|
| GET | `/api/telegram-bots/health` | PyOrchestrator қолжетімділігі |
| GET | `/api/telegram-bots/bots` | Боттар тізімі |
| POST | `/api/telegram-bots/bots` | Жасау |
| PUT | `/api/telegram-bots/bots/:id` | Жаңарту |
| DELETE | `/api/telegram-bots/bots/:id` | Жою |
| POST | `/api/telegram-bots/bots/:id/start` | Іске қосу |
| POST | `/api/telegram-bots/bots/:id/stop` | Тоқтату |
| GET | `/api/telegram-bots/bots/:id/link` | QR және `t.me/...` сілтемесі |
| POST | `/api/telegram-bots/bots/refresh` | Үлгі синхрондауы + restart |
