v1.1.19 — Integrity DATA_DIR fix · Исправление проверки DATA_DIR

**English** · **Русский**

---

## 🇬🇧 English

### Fixed

- **False “suspicious DATA_DIR” warning** — integrity check no longer flags valid external host paths such as `/mnt/hdd/data` or `/var/lib/wash-pro-crm`; warning appears only when `DATA_DIR` mistakenly points inside the container bind mount `/deploy`
- **i18n** — clearer EN/RU message explaining when `DATA_DIR` is actually misconfigured

### Upgrade

```bash
git pull
docker compose up -d --build update-bridge dashboard
```

Set `APP_VERSION=1.1.19` in `.env` if the updater does not bump it automatically.

> **External data disk:** `DATA_DIR=/mnt/hdd/data` (or any absolute host path outside `/deploy`) is **valid** — no integrity warning in v1.1.19+.

Docs: https://wash-pro.github.io/WASH-PRO-CRM/en/

## 🇷🇺 Русский

### Исправлено

- **Ложное предупреждение «Подозрительный DATA_DIR»** — проверка целостности больше не помечает корректные внешние пути на хосте (`/mnt/hdd/data`, `/var/lib/wash-pro-crm` и т.п.); предупреждение только если `DATA_DIR` ошибочно указывает внутрь bind mount контейнера `/deploy`
- **Локализация** — уточнённые тексты EN/RU, когда `DATA_DIR` действительно настроен неверно

### Обновление

```bash
git pull
docker compose up -d --build update-bridge dashboard
```

Укажите `APP_VERSION=1.1.19` в `.env`, если апдейтер не обновил версию автоматически.

> **Внешний диск для данных:** `DATA_DIR=/mnt/hdd/data` (или любой абсолютный путь на хосте вне `/deploy`) — **корректная** конфигурация; предупреждения целостности в v1.1.19+ не будет.

Документация: https://wash-pro.github.io/WASH-PRO-CRM/ru/
