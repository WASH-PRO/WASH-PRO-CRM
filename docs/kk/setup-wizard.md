Бірінші Dashboard кіруінен кейін **баптау шеберіне** (`/setup`) бағыттайды, CRM-де `settings.setup.complete = true` белгісі орнатылмайынша.

## Шебер қадамдары

| Қадам | Мазмұны |
|-------|---------|
| **Бастау** | Процестің қысқаша сипаттамасы |
| **Инфрақұрылым** | API қолжетімділігін тексеру (`/api/health`) |
| **Объект** | Автомойка жасау немесе демо-объектті жою |
| **Посттар** | Сериялық нөмірлер, әр пост үшін **MQTT логин/пароль** |
| **Валюта** | Валюта анықтамалығы, әдепкі валюта |
| **MQTT** | Mosquitto есептерін CRM-мен синхрондау |
| **Анықтамалықтар** | Жұмыс режимдері, жеңілдік түрлері |
| **Дайын** | Шеберді аяқтау |

Аяқталғаннан кейін — қош келдіңіз беті (`/welcome`), содан кейін кәдімгі Dashboard.

## RBAC

| Рөл | Шебер |
|-----|--------|
| **Administrator** | Толық қолжетімділік, шеберді аяқтау |
| **Operator** | Объектілер, посттар, анықтамалықтарды жасау/редакциялау; шеберді аяқтау |
| **Viewer** | Тек қадамдарды көру; CRM-ге кіру үшін танысқанын растау керек |

Шеберді аяқтауға (`complete: true`) **update** құқығы бар пайдаланушылар (Administrator, Operator).

## Демо-деректер

`init-seed` «Орталық» демо-автомойкасын және `WP-POST-001` / `WP-POST-002` посттарын жасайды. Шеберде оларды бір батырмамен жоюға немесе өз объектілеріңізбен ауыстыруға болады.

## Қайта іске қосу

- **Жүйе → Баптау шебері** (Administrator / Operator)
- Тікелей сілтеме: `/setup?restart=1`

## «MQTT» қадамындағы MQTT

**«MQTT синхрондау»** батырмасы `POST /api/crm/post-device/mqtt/sync-users` шақырады:

- Mosquitto `passwd` қайта жасайды (`system` + пост логиндері);
- **ACL** генерациялайды — әр пост тек өз `serialNumber` топиктерінде;
- Mosquitto өзгерістерді автоматты түрде қабылдайды (passwd/ACL қайта жүктеу).

Синхрондау **Посттар** бөлімінде пост сақталғанда да орындалады.

## Күй сақтау

`/api/crm/settings` ішіндегі `setup` баптауы (`setup` кілті):

```json
{
  "complete": true,
  "completedAt": "2026-07-07T12:00:00.000Z",
  "completedBy": { "userId": "…", "login": "admin" },
  "skippedSteps": []
}
```
