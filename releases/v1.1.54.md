v1.1.54 — Preserve MQTT post credentials on form save · Сохранение MQTT-учёток поста при сохранении форм

---

## English

### Fixed

- **Post detail page** — saving name / maintenance / features no longer strips `settings.mqttLogin` / `mqttPassword` (whitelist wipe bug)
- **Posts list** — editing a post no longer silently generates a new MQTT password when the field is empty; empty password is rejected
- **Settings** — blank password/token fields keep previously stored secrets; Mosquitto `sync-users` runs only when `mqtt-broker` settings actually change
- **message-processor** — inbound `settings` messages merge into post settings instead of replacing the whole object

### Upgrade

```bash
git fetch origin && git checkout v1.1.54
# or use Dashboard → Settings → Software updates
docker compose build dashboard message-processor
docker compose up -d dashboard message-processor
```

After upgrade, open the post card and confirm MQTT login/password are still present. If they were lost earlier, set them again and run **Sync MQTT**.

## Русский

### Исправлено

- **Страница поста** — сохранение названия / обслуживания / особенностей больше не затирает `settings.mqttLogin` / `mqttPassword` (баг whitelist)
- **Список постов** — при редактировании не генерируется новый MQTT-пароль молча при пустом поле; пустой пароль отклоняется
- **Настройки** — пустые поля пароля/токена не затирают сохранённые секреты; `sync-users` Mosquitto только при реальном изменении `mqtt-broker`
- **message-processor** — входящие `settings` мержатся в settings поста, а не заменяют объект целиком

### Обновление

```bash
git fetch origin && git checkout v1.1.54
# или Dashboard → Настройки → Обновления ПО
docker compose build dashboard message-processor
docker compose up -d dashboard message-processor
```

После обновления откройте карточку поста и убедитесь, что MQTT логин/пароль на месте. Если учётки уже были стёрты раньше — задайте заново и нажмите **Синхронизировать MQTT**.
