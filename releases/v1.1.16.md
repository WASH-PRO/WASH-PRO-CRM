## 🇬🇧 English

### Added

- **System page** (`/system`) in **Main** sidebar (after Status) — server resources and WASH PRO CRM stack info
- **Application block** — WASH PRO CRM name, version, environment, Docker platform, API runtime (Node.js)
- **Components block** — versions of WASH PRO CRM, Dynamic API, PyOrchestrator (from `update-bridge`)
- **Server metrics** — OS, CPU, hostname, uptime, memory, disk, network interfaces (live refresh 30 s)
- **CPU model fallback** — reads `/proc/cpuinfo` in Docker when `os.cpus()` returns empty model
- **i18n** — full EN/RU labels for the System page

### Changed

- **Main menu** — Overview, Status, **System** (platform status; admin group **System** unchanged)

### Upgrade

```bash
git pull
docker compose up -d --build dashboard dynamic-api
```

Set `APP_VERSION=1.1.16` in `.env` if the updater does not bump it automatically.

Docs: https://wash-pro.github.io/WASH-PRO-CRM/en/

## 🇷🇺 Русский

### Добавлено

- **Страница «Система»** (`/system`) в группе **«Главное»** (после «Состояние») — ресурсы сервера и стек WASH PRO CRM
- **Блок «Приложение»** — WASH PRO CRM: название, версия, окружение, платформа Docker, API runtime (Node.js)
- **Блок «Компоненты»** — версии WASH PRO CRM, Dynamic API, PyOrchestrator (из `update-bridge`)
- **Метрики сервера** — ОС, CPU, hostname, uptime, память, диск, сетевые интерфейсы (обновление каждые 30 с)
- **Определение модели CPU** — чтение `/proc/cpuinfo` в Docker, если `os.cpus()` возвращает пустую модель
- **i18n** — полные подписи EN/RU для страницы «Система»

### Изменено

- **Меню «Главное»** — Обзор, Состояние, **Система** (состояние платформы; группа **«Система»** с уведомлениями и настройками без изменений)

### Обновление

```bash
git pull
docker compose up -d --build dashboard dynamic-api
```

Укажите `APP_VERSION=1.1.16` в `.env`, если апдейтер не обновил версию автоматически.

Документация: https://wash-pro.github.io/WASH-PRO-CRM/ru/
