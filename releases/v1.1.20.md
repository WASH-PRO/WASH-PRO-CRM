v1.1.20 — Dashboard auto-update fix · Исправление автообновления из Dashboard

**English** · **Русский**

---

## 🇬🇧 English

### Fixed

- **Update “resets” immediately** — CRM pull step now uses `git fetch` + `git reset --hard origin/main` instead of `git pull --ff-only`; discards accidental edits to **tracked** files while preserving `.env`, `DATA_DIR`, `docker-compose.override.yml`, and `local/`
- **Failed job invisible in UI** — component card keeps showing failed update progress, error text, and step log (not only history)
- **Build ignored compose overrides** — updater build/seed steps use the same compose file list as `scripts/start.sh` (`override`, Redis, PyOrchestrator)
- **Duplicate `APP_VERSION` in `.env`** — updater writes a single version line after pull

### Changed

- **`scripts/compose-files.sh`** — shared compose `-f` flags for `start.sh` and `update-bridge`

### Upgrade

```bash
git pull
docker compose up -d --build update-bridge dashboard
```

Set `APP_VERSION=1.1.20` in `.env` if needed.

> **Server customizations:** use **untracked** `docker-compose.override.yml` and `local/apply-server-patches.sh` — not edits to tracked repo files.

Docs: https://wash-pro.github.io/WASH-PRO-CRM/en/

## 🇷🇺 Русский

### Исправлено

- **Обновление «сразу сбрасывается»** — шаг pull CRM: `git fetch` + `git reset --hard origin/main` вместо `git pull --ff-only`; сбрасывает случайные правки **отслеживаемых** файлов, сохраняет `.env`, `DATA_DIR`, `docker-compose.override.yml`, `local/`
- **Не видна ошибка обновления** — на карточке компонента остаются прогресс, текст ошибки и лог шага (не только «История»)
- **Сборка без compose override** — шаги build/seed апдейтера используют тот же набор compose-файлов, что и `scripts/start.sh` (override, Redis, PyOrchestrator)
- **Дубли `APP_VERSION` в `.env`** — после pull одна строка версии

### Изменено

- **`scripts/compose-files.sh`** — общие флаги `-f` для `start.sh` и `update-bridge`

### Обновление

```bash
git pull
docker compose up -d --build update-bridge dashboard
```

При необходимости: `APP_VERSION=1.1.20` в `.env`.

> **Серверные правки:** только в **неотслеживаемых** `docker-compose.override.yml` и `local/apply-server-patches.sh` — не в файлах git.

Документация: https://wash-pro.github.io/WASH-PRO-CRM/ru/
