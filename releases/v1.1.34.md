## English

### Added

- **Notifications — software updates** — events: started, success, failed (CRM, Dynamic API, PyOrchestrator)
- **Notifications — modules** — install, uninstall, start, stop, update, errors

### Fixed

- **CRM auto-update** — reliable `modules-bridge` startup: compose hook without `-x` check, `crm-update-ensure-modules-bridge.sh`, ensure on seed/health steps, `--force-recreate`

### Remote site without SSH

1. Open CRM in browser (admin account)
2. **Settings → Software updates** → update to **v1.1.34**
3. Wait for update to finish (health now fails if modules-bridge is down)
4. If Modules still fail: **Settings → Integrity and repair → Rebuild modules-bridge + dashboard**
5. Hard refresh Modules page (Cmd+Shift+R)

## Русский

### Добавлено

- **Уведомления — обновления ПО** — запуск, успех, ошибка (CRM, Dynamic API, PyOrchestrator)
- **Уведомления — модули** — установка, удаление, запуск, остановка, обновление, ошибки

### Исправлено

- **Автообновление CRM** — надёжный запуск `modules-bridge`: hook без `-x`, `crm-update-ensure-modules-bridge.sh`, ensure на шагах seed/health, `--force-recreate`

### Удалённый объект без SSH

1. Откройте CRM в браузере (учётка администратора)
2. **Настройки → Обновления ПО** → обновление до **v1.1.34**
3. Дождитесь завершения (health теперь падает, если modules-bridge не поднялся)
4. Если «Модули» не работают: **Настройки → Диагностика и исправления → Пересобрать modules-bridge + dashboard**
5. Жёсткое обновление страницы «Модули» (Cmd+Shift+R)
