v1.1.26 — Fix version after CRM update · Версия после обновления CRM

**English** · **Русский**

---

## 🇬🇧 English

### Fixed

- **CRM version after update** — `APP_VERSION` is written to `.env` **before** the Build step so Docker build bakes the target version into `version.json` and Vite
- **Dashboard startup** — `version.json` is refreshed from `APP_VERSION` env when the container starts (covers cached image layers)

### Upgrade

```bash
git pull
docker compose up -d --build update-bridge dashboard
```

Docs: https://wash-pro.github.io/WASH-PRO-CRM/en/

## 🇷🇺 Русский

### Исправлено

- **Версия CRM после обновления** — `APP_VERSION` записывается в `.env` **до** шага «Сборка», чтобы в `version.json` и Vite попала целевая версия
- **Старт dashboard** — `version.json` обновляется из `APP_VERSION` при запуске контейнера (если образ был из кэша)

### Обновление

```bash
git pull
docker compose up -d --build update-bridge dashboard
```

Документация: https://wash-pro.github.io/WASH-PRO-CRM/ru/
