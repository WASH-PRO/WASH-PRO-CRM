v1.1.57 — Fix CRM offline while ETH MQTT is connected · CRM офлайн при MQTT OK на ETH

---

## English

### Fixed

- **MQTT `system` password drift** — after reboot, `mosquitto-init` no longer overwrites an existing passwd from `.env`, which used to fight CRM **Settings → MQTT** (`washpro` seed) and leave `message-processor` with `Not authorized` while the ETH panel still showed MQTT OK.
- **Startup heal** — processor upgrades empty/seed `washpro` in `mqtt-broker` settings to `MQTT_PASSWORD`, syncs Mosquitto passwd, then connects (so CRM online status tracks telemetry again).
- **init-seed** — default `mqtt-broker.systemPassword` follows `MQTT_PASSWORD` from the environment when present.

### Upgrade

```bash
# Dashboard → Settings → Software updates → CRM
# or:
git fetch origin && git checkout v1.1.57
docker compose up -d --build message-processor init-seed
```

If the post is still offline: `./scripts/fix-mqtt.sh` then rebuild `message-processor`. Force `system` from `.env` only if needed: `FORCE_MQTT_SYSTEM_PASS=1 ./scripts/fix-mqtt.sh`.

## Русский

### Исправлено

- **Рассинхрон пароля MQTT `system`** — после рестарта `mosquitto-init` больше не перезаписывает существующий passwd из `.env`, из‑за чего processor ловил `Not authorized`, а ETH при этом показывал MQTT OK (seed `washpro` в Настройках ≠ `MQTT_PASSWORD`).
- **Лечение при старте** — processor поднимает пустой/seed `washpro` в `mqtt-broker` до `MQTT_PASSWORD`, синхронизирует passwd и только потом подключается (CRM снова видит телеметрию / онлайн).
- **init-seed** — пароль `mqtt-broker.systemPassword` по умолчанию берётся из `MQTT_PASSWORD`, если задан.

### Обновление

```bash
# Настройки → Обновления ПО → CRM
# или:
git fetch origin && git checkout v1.1.57
docker compose up -d --build message-processor init-seed
```

Если пост всё ещё офлайн: `./scripts/fix-mqtt.sh` и пересборка `message-processor`. Принудительный сброс `system` из `.env` только при необходимости: `FORCE_MQTT_SYSTEM_PASS=1 ./scripts/fix-mqtt.sh`.
