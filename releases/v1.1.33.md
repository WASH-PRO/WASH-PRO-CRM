## English

### Fixed

- **Modules page** — Safari/WebKit error `The string did not match the expected pattern`: JWT decode without `atob`, safe JSON parsing, friendly error message
- **modules-bridge offline** — repair from browser: **Settings → Integrity and repair → Rebuild modules-bridge + dashboard** (no SSH)
- **Auto-update** — modules-bridge health retried 5×; dashboard no longer blocked if modules-bridge fails to start

### Changed

- **Password fields** — show/hide toggle for administrators (Settings, Users, Profile)

### Remote site without SSH

1. Open CRM in browser (any admin account)
2. **Settings → Software updates** → update to **v1.1.33**
3. If Modules still fail: **Settings → Integrity and repair → Check** → enable **Rebuild modules-bridge + dashboard** → **Apply fixes**
4. Hard refresh Modules page (Cmd+Shift+R)

## Русский

### Исправлено

- **Страница «Модули»** — ошибка Safari `The string did not match the expected pattern`: JWT без `atob`, безопасный JSON, понятное сообщение
- **modules-bridge не запущен** — починка из браузера: **Настройки → Диагностика и исправления → Пересобрать modules-bridge + dashboard** (без SSH)
- **Автообновление** — health modules-bridge с 5 повторами; dashboard не блокируется при сбое modules-bridge

### Изменено

- **Поля пароля** — кнопка показа для администраторов (настройки, пользователи, профиль)

### Удалённый объект без SSH

1. Откройте CRM в браузере (учётка администратора)
2. **Настройки → Обновления ПО** → обновление до **v1.1.33**
3. Если «Модули» всё ещё не работают: **Настройки → Диагностика и исправления → Проверить** → **Пересобрать modules-bridge + dashboard** → **Применить**
4. Жёсткое обновление страницы «Модули» (Cmd+Shift+R)
