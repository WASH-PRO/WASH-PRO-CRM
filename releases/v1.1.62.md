v1.1.62 — Telegram API proxy via pyorch-bridge · Прокси Telegram через bridge

---

## English

### Fixed

- **Informational / operator bots not answering** — `pyorch-runtime` sandboxes on some hosts cannot reach `api.telegram.org` (IPv6/egress). Bots now call Telegram through `pyorch-bridge` (`POST /internal/telegram/call`) on the internal Docker network; bridge uses IPv4 HTTPS to Telegram.

### Upgrade

```bash
# Dashboard → Settings → Software updates → CRM
# Then restart Telegram bots
```

## Русский

### Исправлено

- **Боты не отвечают в Telegram** — sandbox `pyorch-runtime` на части хостов не достучаться до `api.telegram.org`. Вызовы Telegram идут через `pyorch-bridge` (`POST /internal/telegram/call`) по внутренней сети; bridge ходит в Telegram по IPv4 HTTPS.

# Затем перезапустите Telegram-ботов
