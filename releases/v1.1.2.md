v1.1.2 — Updater version detection · Определение версии в апдейтере

**English** · **Русский**

---

## 🇬🇧 English

### Fixed

- **Updater did not detect new version** — read `APP_VERSION` from `.env` which was not bumped on releases; comparison with GitHub was wrong; versions synced
- **GitHub API rate limit** — without token (60 req/hour) check silently returned "no updates"; now shows clear reason and supports `GITHUB_TOKEN` (5000/hour)

### Added

- **`GITHUB_TOKEN`** for `update-bridge` — authenticated GitHub API requests
- `error` field in component check — UI shows failure reason (rate limit, network, etc.)

### Upgrade

```bash
git pull
# optional: GITHUB_TOKEN=ghp_xxx in .env
docker compose up -d --build --no-deps update-bridge dashboard
```

> Local `.env` is not auto-updated — set `APP_VERSION=1.1.2` manually if needed.

## 🇷🇺 Русский

### Исправлено

- **Апдейтер не видел новую версию** — `APP_VERSION` в `.env` не обновлялся при релизах; сравнение с GitHub было некорректным
- **Лимит GitHub API** — без токена (60 запросов/час) проверка молча возвращала «нет обновлений»; теперь понятная причина и `GITHUB_TOKEN` (5000/час)

### Добавлено

- **`GITHUB_TOKEN`** для `update-bridge`
- Поле `error` в проверке компонента — UI показывает причину сбоя

### Обновление

```bash
git pull
# опционально: GITHUB_TOKEN=ghp_xxx в .env
docker compose up -d --build --no-deps update-bridge dashboard
```

> `APP_VERSION` в локальном `.env` обновите вручную до `1.1.2` при необходимости.
