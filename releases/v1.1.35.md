v1.1.35 — Fix CRM update health step (Docker paths) · Исправление health при автообновлении

---

## English

### Fixed

- **CRM auto-update failed on health** — health step now receives correct Docker host paths (`composeCommandEnv`); new `crm-update-compose-env.sh` prevents `.env` from overwriting `WASH_HOST_PROJECT_ROOT`
- **compose-files hook** — health/ensure no longer trigger a duplicate full CRM rebuild from update-bridge

### If v1.1.34 update failed (v1.1.34 → v1.1.34)

1. **Settings → Software updates** → update to **v1.1.35**
2. Wait for completion (build + health should pass)
3. Hard refresh `/modules` (Cmd+Shift+R)
4. If still failing: **Settings → Integrity and repair → Rebuild modules-bridge + dashboard**

## Русский

### Исправлено

- **Автообновление CRM падало на health** — шаг health получает корректные пути Docker; `crm-update-compose-env.sh` не даёт `.env` затереть `WASH_HOST_PROJECT_ROOT`
- **Hook compose-files** — health/ensure больше не запускают повторную полную сборку

### Если обновление v1.1.34 завершилось с failed

1. **Настройки → Обновления ПО** → обновить до **v1.1.35**
2. Дождаться завершения
3. Жёсткое обновление `/modules` (Cmd+Shift+R)
4. Если не помогло: **Настройки → Диагностика и исправления → Пересобрать modules-bridge + dashboard**
