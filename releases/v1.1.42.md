## English

### Changed

- **VK Publisher module (catalog v1.2.0)** — publishes **text only** to VK (title + plain text, HTML stripped). `imageUrl` and images are **ignored** for VK; they remain for CRM and the information Telegram bot. Updated module UI, help page, and catalog description
- **modules-bridge** — `SECRET_*` → env bootstrap for wash modules (fixes VK token/group not reaching Python after PyOrch inject); TypeScript build fix; `POST /installed/:id/reregister` endpoint

### Added

- **`scripts/reset-info-messages-vk-demo.mjs`** — delete all Publications and create 10 demo entries (HTML, emoji, images for Telegram/CRM)
- **`scripts/fix-info-message-images.mjs`** — replace blocked image hosts (e.g. picsum.photos) with reachable placeholders

### Documentation

- **docs / wiki** — VK Publisher text-only behavior; version **v1.1.42**; dashboard in-app help (Modules, Publications)

### Upgrade

```bash
git pull
docker compose build --pull=false modules-bridge dashboard
docker compose up -d modules-bridge dashboard
# VK Publisher: Automation → Modules → vk-publisher → Update → Restart
```

Community access token (`wall`, `groups`) is enough for VK Publisher — **photos scope not required**.

## Русский

### Изменено

- **Модуль VK публикатор (каталог v1.2.0)** — во VK уходит **только текст** (заголовок + текст, HTML очищается). `imageUrl` и картинки **не передаются** во VK; они остаются для CRM и информационного Telegram-бота. Обновлены UI модуля, справка и описание в каталоге
- **modules-bridge** — bootstrap `SECRET_*` → env для wash-модулей (исправление: токен/ID группы не доходили до Python); исправление сборки TS; endpoint `POST /installed/:id/reregister`

### Добавлено

- **`scripts/reset-info-messages-vk-demo.mjs`** — удаление всех публикаций и создание 10 демо-записей (HTML, эмодзи, картинки для Telegram/CRM)
- **`scripts/fix-info-message-images.mjs`** — замена недоступных URL картинок (например picsum.photos) на рабочие placeholder

### Документация

- **docs / wiki** — режим «только текст» VK публикатора; версия **v1.1.42**; встроенная справка dashboard (Модули, Публикации)

### Обновление

```bash
git pull
docker compose build --pull=false modules-bridge dashboard
docker compose up -d modules-bridge dashboard
# VK публикатор: Автоматизация → Модули → vk-publisher → Обновить → Перезапустить
```

Для VK публикатора достаточно **ключа доступа сообщества** (`wall`, `groups`) — право **photos** не требуется.
