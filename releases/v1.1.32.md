## 🇬🇧 English

### Fixed

- **Modules page error** (`The string did not match the expected pattern`) — Safari JWT decode (base64url); safe JSON parsing from modules-bridge
- **Auto-update skipped modules-bridge** — CRM update now builds and starts `modules-bridge` automatically (even on older update-bridge images via `scripts/crm-update-build.sh`)

### Changed

- **`scripts/crm-update-build.sh`** — canonical CRM build step (init-seed, modules-bridge, dashboard stack)
- **`scripts/crm-update-health.sh`** — health check includes modules-bridge `:3024`
- **Modules page** — compact filters, icon-only card actions, refresh catalog with label

### Upgrade

Run **Settings → Software updates → Update** for WASH PRO CRM (recommended).

Or manually:

```bash
git pull
docker compose up -d --build modules-bridge dashboard update-bridge
```

Docs: https://wash-pro.github.io/WASH-PRO-CRM/en/modules/

## 🇷🇺 Русский

### Исправлено

- **Ошибка на странице «Модули»** (`The string did not match the expected pattern`) — декодирование JWT в Safari (base64url); безопасный разбор JSON от modules-bridge
- **Автообновление не поднимало modules-bridge** — шаг «Сборка» CRM теперь собирает и запускает `modules-bridge` (в т.ч. на старых образах update-bridge через `scripts/crm-update-build.sh`)

### Изменено

- **`scripts/crm-update-build.sh`** — единый шаг сборки CRM (init-seed, modules-bridge, dashboard)
- **`scripts/crm-update-health.sh`** — проверка здоровья включает modules-bridge `:3024`
- **Страница «Модули»** — компактные фильтры, icon-only на карточках, «Обновить каталог» с текстом

### Обновление

Запустите **Настройки → Обновления ПО → Обновить** для WASH PRO CRM (рекомендуется).

Или вручную:

```bash
git pull
docker compose up -d --build modules-bridge dashboard update-bridge
```

Документация: https://wash-pro.github.io/WASH-PRO-CRM/ru/modules/
