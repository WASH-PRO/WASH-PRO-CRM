## 🇬🇧 English

### Added

- **Integrity & repair wizard** in **Settings** — diagnose paths, `.env`, key files, Docker socket, and update readiness
- **Repair API** in `update-bridge` — `GET/POST /api/crm/updates/repair` (admin JWT)
- **Diagnostics** — `WASH_HOST_PROJECT_ROOT`, `DATA_DIR`, bind-mount detection, critical files/dirs, `docker compose config`, stuck update jobs
- **One-click fixes** — sync host root to `.env`, normalize `DATA_DIR`, `git safe.directory`, clear stuck job, Mosquitto repair (`fix-mqtt.sh`), `init-seed`
- **i18n** — full EN/RU labels for the repair wizard

### Changed

- **`scripts/start.sh`** — includes `docker-compose.override.yml` when present
- **CRM update pull step** — runs `local/apply-server-patches.sh` after `git pull` when executable

### Upgrade

```bash
git pull
docker compose up -d --build update-bridge dashboard
```

Set `APP_VERSION=1.1.17` in `.env` if the updater does not bump it automatically.

If updates fail or paths look wrong: **Settings → Integrity and repair → Check integrity**.

Docs: https://wash-pro.github.io/WASH-PRO-CRM/en/

## 🇷🇺 Русский

### Добавлено

- **Мастер «Целостность и исправление»** в **Настройках** — диагностика путей, `.env`, ключевых файлов, Docker socket и готовности к обновлению
- **Repair API** в `update-bridge` — `GET/POST /api/crm/updates/repair` (JWT администратора)
- **Диагностика** — `WASH_HOST_PROJECT_ROOT`, `DATA_DIR`, определение bind-mount, критичные файлы/каталоги, `docker compose config`, зависшие обновления
- **Исправления в один клик** — синхронизация корня проекта в `.env`, нормализация `DATA_DIR`, `git safe.directory`, сброс зависшего обновления, Mosquitto (`fix-mqtt.sh`), `init-seed`
- **i18n** — полные подписи EN/RU для мастера исправления

### Изменено

- **`scripts/start.sh`** — подключает `docker-compose.override.yml`, если файл есть
- **Шаг pull при обновлении CRM** — после `git pull` запускает `local/apply-server-patches.sh`, если скрипт исполняемый

### Обновление

```bash
git pull
docker compose up -d --build update-bridge dashboard
```

Укажите `APP_VERSION=1.1.17` в `.env`, если апдейтер не обновил версию автоматически.

Если обновление не проходит или сбились пути: **Настройки → Целостность и исправление → Проверить целостность**.

Документация: https://wash-pro.github.io/WASH-PRO-CRM/ru/
