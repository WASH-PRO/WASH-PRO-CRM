## 🇬🇧 English

### Added

- **Workload chart** on Overview — daily usage time line chart below Revenue (from usage stats)
- **Delete all** on Notifications page
- **Demo Telegram bots** on first install (Management / Service / Informational, no tokens, stopped)
- **`POST /api/telegram-bots/bots/stop-all`** — stop all bots with orphan process cleanup
- **`seed-info-messages.mjs`** — richer news/promotion seeding (`COUNT`, `INTERVAL_MIN`, 50/50 mix)

### Fixed

- **Second Telegram bot creation** — PyOrchestrator `ScriptCreate.metadata` (500 Internal Server Error)
- **Telegram bots list** — bots disappearing after start/stop until page reload
- **Ghost bot** — orphan sandbox process after stop; reliable runtime kill on script stop
- **Dual polling race** — shared poll lock, staggered restarts, stop grace period
- **Informational bot occupancy v2.2** — post is **free** only in `program_9` (payment entry); all other modes = busy

### Changed

- **Bot stop** — wait for full shutdown, `deleteWebhook`, disable script
- **pyorch-bridge** — no mass bot restart on bridge container boot

### Upgrade

```bash
git pull
docker compose -f docker-compose.yml -f docker-compose.pyorchestrator.yml up -d --build \
  dashboard init-seed pyorch-backend pyorch-runtime pyorch-bridge
```

Set `APP_VERSION=1.1.15` in `.env` if the updater does not bump it automatically.

Docs: https://wash-pro.github.io/WASH-PRO-CRM/en/

## 🇷🇺 Русский

### Добавлено

- **График загруженности** на Обзоре — по дням, под графиком «Выручка» (из статистики использования)
- **Удалить все** на странице «Уведомления»
- **Демо Telegram-боты** при первой установке (Управление / Сервисный / Информационный, без токенов, остановлены)
- **`POST /api/telegram-bots/bots/stop-all`** — остановка всех ботов с очисткой зависших процессов
- **`seed-info-messages.mjs`** — улучшенная генерация новостей и акций (`COUNT`, `INTERVAL_MIN`, микс 50/50)

### Исправлено

- **Создание второго Telegram-бота** — `ScriptCreate.metadata` в PyOrchestrator (500)
- **Список ботов Telegram** — пропадание записей после запуска/остановки до перезагрузки страницы
- **«Фантомный» бот** — осиротевший процесс sandbox после stop; надёжное завершение в runtime
- **Гонка двух ботов** — общий poll lock, поочерёдный restart, пауза после stop
- **Занятость в информационном боте v2.2** — **свободен** только `program_9` (внесение оплаты); остальные режимы = занят

### Изменено

- **Остановка ботов** — ожидание полного завершения, `deleteWebhook`, disable скрипта
- **pyorch-bridge** — без массового перезапуска ботов при старте контейнера

### Обновление

```bash
git pull
docker compose -f docker-compose.yml -f docker-compose.pyorchestrator.yml up -d --build \
  dashboard init-seed pyorch-backend pyorch-runtime pyorch-bridge
```

Укажите `APP_VERSION=1.1.15` в `.env`, если апдейтер не обновил версию автоматически.
